# Практическая задача: командная работа с Git

## Цель

Создать небольшой Python-проект, вести его разработку в нескольких ветках и отправить историю изменений в удалённый репозиторий.

## Проект

Создай консольную программу для анализа списка продаж.

Программа должна уметь:

* хранить список продаж;
* подсчитывать общую сумму;
* находить максимальную продажу;
* выводить количество продаж.

Пример данных:

```python
sales = [1200, 850, 430, 2100, 1750]
```

---

## Этап 1. Создание проекта

1. Создай папку проекта:

```text
sales-analyzer
```

2. Перейди в неё.

3. Создай файл:

```text
main.py
```

4. Инициализируй Git-репозиторий.

5. Проверь состояние репозитория.

6. Добавь файл `main.py` в staging area.

7. Создай первый commit с сообщением:

```text
Создал структуру проекта
```

---

## Этап 2. Первая версия программы

В файле `main.py` создай список продаж и выведи его:

```python
sales = [1200, 850, 430, 2100, 1750]

print(sales)
```

После изменения:

1. Проверь статус Git.
2. Добавь изменённый файл в staging area.
3. Создай commit:

```text
Добавил список продаж
```

---

## Этап 3. Работа с веткой `feature/total`

1. Создай новую ветку:

```text
feature/total
```

2. Перейди в неё.

3. Добавь функцию подсчёта общей суммы продаж:

```python
def calculate_total(sales):
    ...
```

4. Вызови функцию и выведи результат.

5. Проверь статус.

6. Создай commit:

```text
Добавил расчёт общей суммы продаж
```

---

## Этап 4. Работа с веткой `feature/max-sale`

1. Вернись в основную ветку.

2. Создай новую ветку и сразу перейди в неё:

```text
feature/max-sale
```

3. Добавь функцию поиска максимальной продажи:

```python
def find_max_sale(sales):
    ...
```

4. Вызови функцию и выведи результат.

5. Используй команду, которая одновременно добавляет изменения уже отслеживаемого файла и создаёт commit.

Сообщение commit:

```text
Добавил поиск максимальной продажи
```

---

## Этап 5. Слияние веток

1. Перейди в основную ветку.

2. Объедини с ней ветку:

```text
feature/total
```

3. Затем объедини ветку:

```text
feature/max-sale
```

4. Запусти программу и проверь, что обе функции работают.

5. Выведи список веток.

---

## Этап 6. Добавление новой функции

В основной ветке добавь функцию подсчёта количества продаж:

```python
def count_sales(sales):
    ...
```

Вызови функцию и выведи результат.

Создай commit:

```text
Добавил подсчёт количества продаж
```

---

## Этап 7. Работа с удалённым репозиторием

1. Создай пустой репозиторий на GitHub, GitLab или Bitbucket:

```text
sales-analyzer
```

2. Свяжи локальный репозиторий с удалённым.

Для имени удалённого репозитория используй:

```text
origin
```

3. Отправь основную ветку в удалённый репозиторий.

4. Проверь, что файл и commits появились на сайте.

---

## Этап 8. Работа с `pull`

1. Через интерфейс GitHub, GitLab или Bitbucket измени файл `main.py`.

Например, добавь комментарий:

```python
# Программа для анализа продаж
```

2. В локальном репозитории стяни изменения из удалённой основной ветки.

3. Проверь содержимое `main.py`.

4. Проверь статус Git.

---

## Этап 9. Работа с `clone`

1. Перейди в другую директорию, не внутрь текущего проекта.

2. Склонируй удалённый репозиторий.

3. Перейди в склонированную папку.

4. Проверь:

* список файлов;
* список веток;
* статус Git;
* работоспособность программы.

---

git merge

git push

git clone
```

Учебный Python-проект для практики Git.

- находить максимальную продажу;

```

3. Проверь статус Git.

4. Добавь `README.md` в staging area.

5. Создай commit:

```text
Добавил описание проекта
```

6. Отправь commit в удалённый репозиторий.

---

### Работа с `.gitignore`

1. Создай файл:

```text
.gitignore
```

2. Добавь в него:

```gitignore
__pycache__/
.venv/
.env
```

3. Создай директорию:

```text
__pycache__
```

4. Создай файл:

```text
.env
```

5. Выполни:

```bash
git status
```

6. Убедись, что Git не предлагает добавить `__pycache__` и `.env`.

7. Добавь `.gitignore` в staging area.

8. Создай commit:

```text
Добавил исключения Git
```

9. Отправь изменения в удалённый репозиторий.

---

## Обязательные условия

* Нельзя выполнять все изменения в основной ветке.
* Минимум две функции должны быть реализованы в отдельных ветках.
* Перед каждым commit необходимо выполнять:

```bash
git status
```

* Перед отправкой проекта в удалённый репозиторий программа должна запускаться без ошибок.
* Нельзя добавлять `.env` в репозиторий.
